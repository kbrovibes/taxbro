## Spec format

Each spec includes:
- **Problem** — what gap or limitation is being addressed
- **Goals / Non-goals** — scope boundaries
- **Decision** — the chosen approach and why
- **Technical design** — concrete file layout, protocols, interfaces
- **Migration** — how to move from current state to proposed state
- **Open questions** — unresolved decisions or risks
